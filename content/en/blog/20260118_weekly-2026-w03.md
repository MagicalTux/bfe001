Week of January 12–18. One major new repo, and continued small work
on older ones.

## cterm

**[cterm](https://github.com/KarpelesLab/cterm)** — a new terminal
emulator, seeded on the 13th, 74 commits in five days. Pure Rust
with platform-native UI: AppKit + CoreGraphics on macOS, GTK4 on
Linux, Win32 on Windows. Custom VT100/ANSI engine with 24-bit color,
256-color fallback, proper wide / combining / emoji character
handling, regex-searchable scrollback, and a modular architecture
designed around running AI coding assistants like Claude Code (and,
per the README, "ClaudeTerm" in the name) as first-class workloads
rather than afterthoughts.

The question "what does a terminal need to be, if half your sessions
are running an AI inside them?" doesn't get the same answer as
"general-purpose terminal". This week is mostly the foundation — VT
engine, buffer management, platform shells — but the framing is
deliberate.
