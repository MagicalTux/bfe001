Week of March 30 – April 5. A lot of what I publish starts from "why
does Go not have this?" This week, six repos push in that direction.

## Filling Go's gaps

- **[gopcre2](https://github.com/KarpelesLab/gopcre2)** — PCRE2 in
  pure Go, no cgo. Backreferences, lookahead / lookbehind, atomic
  groups, recursive patterns, Unicode properties. Go's stdlib `regexp`
  uses RE2 for guaranteed linear time and deliberately omits these
  features; `gopcre2` gives you the features RE2 leaves out, with an
  API similar to the standard library.
- **[gobzip2](https://github.com/KarpelesLab/gobzip2)** — pure-Go
  bzip2 *compression* as well as decompression. The standard library's
  `compress/bzip2` is decompression-only; this closes the gap, using
  the bzip2 1.0.8 reference C implementation as a spec.
- **[gomailparse](https://github.com/KarpelesLab/gomailparse)** —
  streaming MIME parser. Reads from an `io.Reader` in a single pass
  and stores only headers and byte offsets — the body is never
  buffered. Built for indexing very large messages and locating
  attachment ranges for on-demand random access.
- **[gotz](https://github.com/KarpelesLab/gotz)** — raw IANA timezone
  data access in Go: transitions, zone types, POSIX TZ rules,
  embedded `tzdata`. Sits deliberately under the standard `time`
  package's abstraction for when you need the underlying data
  directly.
- **[pktkit](https://github.com/KarpelesLab/pktkit)** — zero-copy
  L2/L3 packet handling. `Frame` and `Packet` are `[]byte` aliases
  with typed header accessors — no wrapper allocation, no copies on
  the hot path. Seeded on April 4 and ran up 35 commits by the end of
  the week laying down IPv4 + IPv6 primitives.
- **[aipencil](https://github.com/KarpelesLab/aipencil)** — CLI that
  renders structured JSON scene descriptions to SVG and PNG. The idea:
  give AI systems a deterministic, cheap way to produce images by
  declaring content and relationships, not sizes and coordinates.
  Same input always produces the same image. No GPU. First full week
  of activity after seeding on April 2 — 18 commits.

## Sandboxing + networking

Two networking-adjacent projects that fit together more than the
topic suggests:

**[bnpm](https://github.com/KarpelesLab/bnpm)** — "Bubble NPM" —
sandboxes package-manager commands inside Linux namespaces. Mount
namespace with `pivot_root` restricts the filesystem to the project
directory (read-write) plus read-only system paths. An empty network
namespace gives build/test commands zero network access; install
commands go through a userspace TAP proxy that filters DNS and IPs
to allowed registries only. No root required — unprivileged user
namespaces do the work. The goal is to make a malicious `install`
script physically unable to reach outside the project tree.

**[slirp](https://github.com/KarpelesLab/slirp)** — user-mode NAT
and IP stack in Go, 28 commits this week. Same intent as the
original SLIRP: give a process TCP/IP connectivity without root,
`CAP_NET_ADMIN`, or a TUN device. Useful for VM networking,
containers, network testing, and anything that wants a protocol
stack that runs fully in userspace.

## PHP in Go, PHP in Rust

**[goro](https://github.com/KarpelesLab/goro)** got 128 commits and
the parallel Rust port **[goro-rs](https://github.com/KarpelesLab/goro-rs)**

conformance.
